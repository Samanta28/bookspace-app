# AGENTS.md

Projekt realizowany indywidualnie (za zgodą prowadzącego).

## Organizacja pracy

* brak bezpośrednich pushy do gałęzi `main`
* każda zmiana realizowana poprzez Pull Request
* workflow: `feature → develop → main`

## Struktura projektu

* **Auth Service**
  odpowiedzialny za rejestrację, logowanie i generowanie tokenów JWT

* **Book Service**
  odpowiedzialny za zarządzanie książkami, recenzjami oraz listą „do przeczytania”

## Zakres odpowiedzialności

Projekt wykonany w całości przez jedną osobę:

* backend (FastAPI)
* projekt bazy danych
* implementacja API
* zarządzanie repozytorium i Git workflow
